# 复习建议与资源（小林 coding）

> 资料来源：[小林 coding](https://xiaolincoding.com/)

---

## 1. 面试结构

后端技术面试主要考察 **三大方向**：

```
八股文（基础知识） + 项目经历 + 算法题
```

- **八股**：考察基础是否扎实，能否讲清原理
- **项目**：考察实战能力和问题解决能力
- **算法**：考察编码能力和逻辑思维

## 3. 学习方法

### 3.1 先体系后刷题

```
图解系列（建立框架）→ 面试题合集（针对性刷题）→ 大厂面经（验证热点）
```

不要上来就背零散面经，先理解知识体系，再看面经会事半功倍。

### 3.2 以点扩面

一个面试题关联一串知识点。例如：
- 「HashMap 线程不安全」→ synchronized → ConcurrentHashMap → CAS → AQS
- 「MySQL 可重复读」→ MVCC → Read View → undo log → 间隙锁

### 3.3 画图辅助记忆

以下知识点强烈建议手绘：
- B+ 树结构
- TCP 三次握手 / 四次挥手
- HashMap put 流程
- Spring Bean 生命周期
- JVM 内存结构
- MVCC 版本链
- Redis 主从复制

### 3.4 模拟面试

1. 自问自答，限时 3 分钟/题
2. 录音复盘，检查表达是否清晰
3. 找同学互相面试
4. 投递前集中过一遍高频题

## 4. 从零开始学习路线（含 LeetCode）

> 适用：计算机相关专业在校生、转行/自学后端、距离面试 **3~6 个月**。
> 原则：**八股建立体系 + 算法每天动手 + 项目贯穿始终**，三者并行，不要等八股背完再刷题。

### 4.1 整体节奏

```
阶段 0（1 周）   环境 + Java 语法 + 极简算法
阶段 1（3~4 周） Java 基础/集合 + 数据结构 + Easy 题
阶段 2（3~4 周） 并发/JVM + 计网/OS + Easy/Medium 题
阶段 3（4~5 周） Spring + MySQL + Redis + Medium 题
阶段 4（3~4 周） MQ/分布式/系统设计 + Hot 100 冲刺
阶段 5（2 周）   面经复盘 + 模拟面试 + CodeTop 查漏补缺
```

**每日时间参考（在职 2h / 全职 6h）**：

| 时段 | 在职 | 全职 |
|------|------|------|
| 八股/图解 | 60 min | 3 h |
| LeetCode | 40 min（1 题） | 2 h（2~3 题） |
| 项目/总结 | 20 min | 1 h |

**验收**：能解释「八股 + 项目 + 算法」各占多少；LeetCode 能独立提交通过 2 题 Easy。

### 4.8 LeetCode 刷题规范

**难度比例（全程）**：

| 阶段 | Easy | Medium | Hard |
|------|------|--------|------|
| 1~2 | 70% | 30% | 0% |
| 3~4 | 30% | 65% | 5% |
| 5 | 10% | 80% | 10% |

**时间限制**：

| 难度 | 目标时间 | 超时处理 |
|------|----------|----------|
| Easy | 15~20 min | 看题解，标记「需复习」 |
| Medium | 25~35 min | 看思路不看代码，自己实现 |
| Hard | 45 min | 面试很少考，了解思路即可 |

**错题本格式**（Notion/飞书/本地 md）：

```markdown
## 3. 无重复字符的最长子串
- 标签：滑动窗口、哈希
- 第一次：2026-xx-xx，超时，看了题解
- 核心：右扩左缩，Set 存窗口内字符
- 复习：✅ 第2天 AC  ✅ 第7天 AC
```

**不要做的事**：
- ❌ 按题号从 1 刷到 3000（效率极低）
- ❌ 只看不写，或只复制题解不默写
- ❌ 一题刷过就永不再看（面试前至少复习 2 遍）
- ❌ 八股背完再刷题（算法手感会凉）

## 8. 面试心态

1. **不要慌**：面经里很多题重复，高频题就那些
2. **不要死背**：理解原理，用自己的话讲出来
3. **不会就说不会**：诚实 + 说思路，比瞎编好
4. **准备项目**：八股过了还要能打，项目是你的差异化
5. **坚持长期主义**：每天进步一点，积累质变

> 祝面试顺利！
